Getting Started

TuenSSO is a standalone SSO server built with Java, providing OpenID Connect services for your applications. It replaces Keycloak with a lightweight, fully controllable codebase.

Architecture

TuenSSO acts as an Identity Provider. Any application that supports OIDC can connect to TuenSSO for authentication.

App A  --OIDC-->
App B  --OIDC-->  TuenSSO :8080
App C  --OIDC-->

Build & Run

Prerequisites

  • Java 21
  • Maven 3.9+
  • Internet connection (first run downloads Node/NPM for frontend build)

Option 1: Dev mode (recommended)

mvn spring-boot:run

This builds the Angular frontend, copies it to classpath resources, and starts Spring Boot.

Option 2: Build artifact

mvn clean package
java -jar target/tuensso-0.0.1-SNAPSHOT.jar

To use a different port:

java -jar target/tuensso-0.0.1-SNAPSHOT.jar --server.port=8081

Option 3: Docker

# Build image
docker build -t tuensso .

# Run container
docker run -p 8080:8080 tuensso

With custom port and PostgreSQL:

docker run -p 8080:8080 \
  -e SPRING_DATASOURCE_URL=jdbc:postgresql://db:5432/tuensso \
  -e SPRING_DATASOURCE_USERNAME=tuensso \
  -e SPRING_DATASOURCE_PASSWORD=secret \
  tuensso

Admin REST API

All admin endpoints require authentication with ROLE_ADMIN.

Clients

MethodPathDescription
GET/api/admin/clientsList all clients
GET/api/admin/clients/{clientId}Get client details
POST/api/admin/clientsRegister new client
PUT/api/admin/clients/{clientId}Update client

Users

MethodPathDescription
GET/api/admin/usersList all users
POST/api/admin/usersCreate user

PUT/api/admin/users/{id}/enableEnable user
PUT/api/admin/users/{id}/disableDisable user
PUT/api/admin/users/{id}/passwordReset password
DELETE/api/admin/users/{id}Delete user

Integrate with Your App

Spring Boot

Add the OAuth2 client dependency:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-oauth2-client</artifactId>
</dependency>

Configure application.yml:

spring:
  security:
    oauth2:
      client:
        registration:
          tuensso:
            client-id: myapp
            client-secret: my-secret
            scope: openid,profile,email
            redirect-uri: "{baseUrl}/login/oauth2/code/{registrationId}"
        provider:
          tuensso:
            issuer-uri: http://localhost:8080

Any OIDC-compatible app

Issuer URI:        http://localhost:8080
Authorization URI: http://localhost:8080/oauth2/authorize
Token URI:         http://localhost:8080/oauth2/token
JWKS URI:          http://localhost:8080/oauth2/jwks
UserInfo URI:      http://localhost:8080/userinfo

PostgreSQL (Production)

By default TuenSSO uses H2 in-memory database. For production, switch to PostgreSQL:

spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/tuensso
    username: tuensso
    password: your-password
    driver-class-name: org.postgresql.Driver
  jpa:
    hibernate:
      ddl-auto: validate

Dockerfile

The project includes a multi-stage Dockerfile:

# Build
docker build -t tuensso .

# Run with H2 (dev)
docker run -p 8080:8080 tuensso

# Run with PostgreSQL (production)
docker run -p 8080:8080 \
  -e SPRING_DATASOURCE_URL=jdbc:postgresql://host:5432/tuensso \
  -e SPRING_DATASOURCE_USERNAME=tuensso \
  -e SPRING_DATASOURCE_PASSWORD=secret \
  -e SPRING_DATASOURCE_DRIVER_CLASS_NAME=org.postgresql.Driver \
  -e SPRING_JPA_HIBERNATE_DDL_AUTO=validate \
  tuensso
